# Система скоринга клиентов Magnetto

## Задача

У девелопера Magnetto средний цикл сделки ~70 дней. За это время клиент заходит на сайт несколько раз, выполняет разные действия (смотрит квартиры, кликает по телефону, заполняет формы), приходит из разных каналов (реклама, органика, прямые заходы). Только ~0.09% клиентов доходят до CRM-сделки (569 из 632K).

Задача системы — на основе поведения прошлых клиентов понять:
1. Какие действия на каком этапе увеличивают вероятность сделки?
2. Какие цепочки каналов приводят к конверсии?
3. Кто из текущих активных клиентов ближе всего к сделке и что с ним делать?

## Почему не ML

При 569 положительных примерах на 632K выборку (0.09%) любая ML-модель будет переобучаться. Статистический подход через lift-анализ надёжнее и объясним: мы не строим "чёрный ящик", а вычисляем конкретные коэффициенты, которые маркетолог может проверить и понять.

## Три витрины и как они связаны

```
dm_step_goal_impact          dm_path_templates
(какие цели работают)        (какие пути работают)
        │                            │
        │   lift по целям            │   CR% по паттернам
        │                            │
        └──────────┐    ┌────────────┘
                   ▼    ▼
          dm_active_clients_scoring
          (кто горячий и что делать)
```

### Уровень 1: dm_step_goal_impact (фундамент)

Отвечает на вопрос: **"Если клиент на N-м визите сделал X — насколько выше его шанс стать CRM-сделкой?"**

Механизм: берём всех клиентов, которые дошли до визита N. Делим на две группы — те, кто выполнил цель X, и те, кто нет. Считаем конверсию каждой группы. Делим одну на другую — получаем lift.

Lift > 1 = цель помогает конверсии. Lift < 1 = цель отрицательно коррелирует с конверсией.

Пример из реальных данных:
- "Отправка формы ипотека" на 1-м визите: lift **284** (25% конвертеров vs 0.09% базовая)
- "Прошел квиз" на 1-м визите: lift **19.6** (слабый сигнал, квиз почти не работает)

### Уровень 2: dm_path_templates (контекст)

Отвечает на вопрос: **"Какие последовательности каналов приводят к сделке и сколько это стоит?"**

Берёт дедуплицированный путь каналов каждого клиента (например, ['organic', 'ad', 'organic']), группирует одинаковые пути, считает конверсию и оценивает стоимость через средний CPC по Директу.

Примеры из реальных данных:
- ['organic', 'ad', 'organic'] → CR 4.98% при стоимости 66 руб. за конверсию
- Пути, оканчивающиеся на ad без organic → 0% конверсии

Вывод: organic в пути — критический фактор; реклама работает как подогрев, но не как финальный канал.

### Уровень 3: dm_active_clients_scoring (продукт)

Отвечает на вопрос: **"Кому показать рекламу сегодня, про какой продукт и через сколько дней?"**

Для каждого активного неконвертированного клиента (визит за последние 90 дней, нет CRM-сделки):
1. Смотрит, какие цели клиент уже выполнял на каких шагах
2. Для каждой пары (шаг, цель) берёт lift из dm_step_goal_impact
3. Суммирует все lift'ы — это **lift_score** клиента
4. Определяет следующий шаг клиента (total_visits + 1)
5. Находит цель с максимальным lift на этом шаге — это **рекомендация** (что тригерить)
6. Берёт медиану gap'а между визитами у конвертеров на этом шаге — это **тайминг** (когда показать рекламу)
7. Присваивает приоритет: hot / warm / cold

## Логика приоритетов

```
HOT  = (есть лид И визит ≤ 7 дней назад) ИЛИ (lift_score > 100 И визит ≤ 3 дней назад)
WARM = (lift_score > 20 И визит ≤ 14 дней назад) ИЛИ (lift_score > 0 И визит ≤ 3 дней назад)
COLD = все остальные
```

Текущее распределение: ~400 hot, ~9K warm, ~358K cold.

## Цепочка обновления

Все витрины обновляются автоматически раз в день по цепочке зависимостей:

```
03:00  dm_direct_performance    (сырые данные Директа)
04:00  dm_client_profile         (профили клиентов)
05:00  dm_client_journey         (визиты клиентов)
05:00  dm_funnel_velocity        (скорость воронки)
06:00  dm_step_goal_impact       (lift целей)
06:00  dm_conversion_paths       (пути конверсии)
07:00  dm_path_templates         (паттерны путей)
08:00  dm_active_clients_scoring (скоринг)
```

Каждая витрина ждёт завершения своих зависимостей через OFFSET в REFRESH.

## Как AI-агент работает с системой

### Стратегические решения → dm_step_goal_impact + dm_path_templates

"Какие цели стимулировать в рекламе?" → step_goal_impact (высокий lift)
"Через какие каналы вести клиента?" → path_templates (высокий CR%)
"Квиз работает или деньги впустую?" → step_goal_impact (lift квиза на каждом шаге)
"Нужна ли organic-прослойка в пути?" → path_templates (сравнить пути с/без organic)

### Операционные решения → dm_active_clients_scoring

"Кому показать рекламу сегодня?" → scoring WHERE priority = 'hot'
"Что именно показать клиенту X?" → recommended_goal_name (цель с макс. lift на его шаге)
"Через сколько дней ретаргетить?" → optimal_retarget_days
"Список client_id для кампании ретаргетинга по costura-town?" → scoring WHERE priority IN ('hot','warm') AND last_project = 'costura-town'
