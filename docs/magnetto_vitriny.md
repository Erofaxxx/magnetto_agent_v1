## dm_traffic_performance

**Гранулярность:** 1 строка = 1 день × проект × канал × устройство × город
**Обновление:** `REFRESH EVERY 1 DAY OFFSET 3 HOUR`
**Назначение:** ежедневная агрегация трафика и всех целей по всем измерениям. Основная витрина для анализа каналов, проектов, воронки дня. Фундамент всей аналитики.

### Размерности

| Поле | Тип | Описание |
|---|---|---|
| `date` | Date | Дата визита |
| `project_slug` | String | Проект из URL входа. Извлекается regex из `startURL`: `/our-projects/([a-z0-9-]+)`. Если визит начался не со страницы проекта — значение `'site'` |
| `start_url` | String | Чистый URL входа без UTM-параметров (для проверки корректности `project_slug`) |
| `utm_source` | String | UTM Source из параметров URL |
| `utm_medium` | String | UTM Medium |
| `utm_campaign` | String | UTM Campaign |
| `traffic_source` | String | Тип трафика из Метрики: `ad`, `organic`, `direct`, `referral`, `internal`, `messenger`, `social`, `recommend`, `saved`, `undefined`. Использовать когда `utm_source` пустой |
| `search_engine` | String | Поисковик для органики: `yandex`, `google`, `bing`, `duckduckgo`, `rambler`. Пустой для неорганики |
| `device_category` | String | Тип устройства: `desktop`, `mobile`, `tablet` |
| `region_city` | String | Город пользователя |

### Трафик

| Поле | Тип | Описание |
|---|---|---|
| `visits` | UInt32 | Количество сессий |
| `new_users` | UInt32 | Новые пользователи (первый визит на сайт) |
| `bounces` | UInt32 | Сессии с одной страницей |
| `total_duration_sec` | UInt64 | Суммарное время на сайте в секундах. Делить на `visits` для получения среднего |
| `total_page_views` | UInt32 | Суммарное количество просмотров страниц |

### Важные особенности

- **`project_slug = 'site'`** — визит начался не со страницы ЖК (главная, блог, контакты, страница квартиры `/property/`, ипотека и т.д.). Это нормально — большая часть трафика входит через главную или рекламные лендинги.
- **Пустой `utm_source`** — не означает органику. Использовать `traffic_source` для определения типа: `organic` = поиск, `direct` = прямой заход, `ad` = реклама без UTM-разметки, `undefined` = не определено.
- **`goal_402733217` (Мусорный трафик)** — флаг спама/ботов. При расчёте CR рекомендуется фильтровать строки где этот счётчик > 0.
- **Производные метрики считать в запросе:** `avg_duration = total_duration_sec / visits`, `bounce_rate = bounces / visits`, `CR = goal_314553735 / visits`.

## dm_client_profile

**Гранулярность:** 1 строка = 1 уникальный клиент, всё время наблюдения
**Обновление:** `REFRESH EVERY 1 DAY OFFSET 4 HOUR`
**Назначение:** сводный портрет каждого посетителя сайта. Ключевая витрина для анализа длинного цикла сделки: сколько касаний до лида, какой first/last touch канал, какие проекты смотрел, как долго «грелся». Все поля — агрегаты по всей истории клиента.

**Фильтрация источника:** только строки с `clientID > 0` (исключены боты и визиты без cookie).

### Поля

| Поле | Тип | Описание |
|---|---|---|
| `client_id` | UInt64 | Идентификатор клиента Яндекс Метрики (clientID) |
| `first_visit_date` | Date | Дата первого визита на сайт |
| `last_visit_date` | Date | Дата последнего визита |
| `days_active` | UInt16 | Количество дней между первым и последним визитом. 0 = был только один день |
| `total_visits` | UInt32 | Суммарное количество сессий за всё время |
| `total_duration_sec` | UInt64 | Суммарное время на сайте в секундах |
| `total_page_views` | UInt32 | Суммарное количество просмотренных страниц |
| `total_bounces` | UInt32 | Количество сессий с одной страницей |
| `projects_visited` | Array(String) | Массив уникальных проектов, которые посещал клиент. Только slug-значения, `'site'` исключён. Порядок не гарантирован |
| `projects_count` | UInt8 | Количество уникальных проектов = `length(projects_visited)` |
| `first_project` | String | Первый проект, страницу которого открыл клиент. Пустая строка если клиент никогда не был на странице `/our-projects/` |
| `last_project` | String | Последний проект, страницу которого открыл клиент |
| `first_traffic_source` | String | `traffic_source` первого визита |
| `first_search_engine` | String | Поисковик первого визита (если органика) |
| `first_utm_source` | String | UTM Source первого визита |
| `first_utm_medium` | String | UTM Medium первого визита |
| `first_utm_campaign` | String | UTM Campaign первого визита |
| `last_traffic_source` | String | `traffic_source` последнего визита |
| `last_utm_source` | String | UTM Source последнего визита |
| `last_utm_medium` | String | UTM Medium последнего визита |
| `last_utm_campaign` | String | UTM Campaign последнего визита |
| `last_device_category` | String | Устройство последнего визита |
| `last_region_city` | String | Город последнего визита |
| `has_lead` | UInt8 | 1 = хотя бы в одном визите сработала цель 314553735 «Все лиды», 0 = нет |
| `first_lead_date` | Date | Дата первого визита с лидом (goal 314553735). `1970-01-01` если лида не было |
| `first_call_date` | Date | Дата первого звонка (goals 63191746, 201619840, 201619843, 201619846). `1970-01-01` если звонка не было |
| `first_form_date` | Date | Дата первой отправки формы (goals 313904512, 314247991, 314247265). `1970-01-01` если формы не было |
| `days_to_first_lead` | Int16 | Количество дней от первого визита до первого лида. `-1` если лида не было |
| `total_lead_visits` | UInt32 | Количество сессий, в которых сработала цель «Все лиды» (клиент мог конвертироваться несколько раз) |
| `has_crm_created` | UInt8 | 1 = клиент попал в CRM как заказ (goal 332069613) |
| `crm_created_date` | Date | Дата первого CRM-заказа. `1970-01-01` если не было |
| `has_crm_paid` | UInt8 | 1 = сделка оплачена (goal 332069614) |
| `crm_paid_date` | Date | Дата первой оплаты. `1970-01-01` если не было |

### Важные особенности

- **Дата `1970-01-01`** в полях `first_lead_date`, `first_call_date`, `first_form_date`, `crm_created_date`, `crm_paid_date` означает «событие не произошло». При фильтрации всегда добавлять `AND first_lead_date != '1970-01-01'` или проверять `has_lead = 1`.
- **`days_to_first_lead = -1`** означает «лида не было». При расчёте среднего цикла фильтровать: `WHERE days_to_first_lead >= 0`.
- **`first_project` / `last_project`** — пустая строка если клиент никогда не открывал страницу `/our-projects/`. Таких клиентов много — они заходили на главную или напрямую на страницу квартиры.
- **`projects_visited`** — порядок элементов в массиве не отражает хронологию посещений. Для хронологии использовать `dm_client_journey`.
- **Атрибуция:** для анализа «какой канал первым приводит покупателей» — `first_utm_source` + `WHERE has_crm_paid = 1`. Для «с какого канала пришли перед сделкой» — `last_utm_source` + `WHERE has_crm_paid = 1`.

## dm_client_journey

**Гранулярность:** 1 строка = 1 сессия 1 клиента
**Обновление:** `REFRESH EVERY 1 DAY OFFSET 5 HOUR`
**Назначение:** пошаговый путь каждого клиента на сайте. Используется для CJM-анализа: на каком визите конвертируются, какой канал «догревает» перед лидом, как чередуются проекты, интервалы возврата. В отличие от `dm_client_profile` здесь каждый визит — отдельная строка с полным контекстом.

**Фильтрация источника:** только строки с `clientID > 0`.

### Поля

| Поле | Тип | Описание |
|---|---|---|
| `client_id` | UInt64 | Идентификатор клиента |
| `visit_id` | UInt64 | Уникальный ID сессии (visitID из Метрики). Позволяет при необходимости уйти в сырые логи `visits_all_fields` за деталями |
| `date` | Date | Дата сессии |
| `visit_datetime` | DateTime | Точное время начала сессии |
| `visit_number` | UInt32 | **Порядковый номер этой сессии у клиента** (1 = первый визит, 2 = второй и т.д.). Вычисляется как `ROW_NUMBER() OVER (PARTITION BY client_id ORDER BY visit_datetime, visit_id)` |
| `total_client_visits` | UInt32 | Всего сессий у этого клиента за всё время наблюдения. Одинаковое для всех строк одного клиента |
| `visit_duration_sec` | UInt32 | Длительность сессии в секундах |
| `page_views` | UInt32 | Количество страниц в сессии |
| `bounce` | UInt8 | 1 = сессия с одной страницей |
| `project_slug` | String | Проект из URL входа этой сессии. `'site'` если вход не со страницы проекта |
| `traffic_source` | String | Тип трафика этой сессии |
| `search_engine` | String | Поисковик (для органики) |
| `utm_source` | String | UTM Source этой сессии |
| `utm_medium` | String | UTM Medium |
| `utm_campaign` | String | UTM Campaign |
| `device_category` | String | Устройство |
| `region_city` | String | Город |
| `days_since_first_visit` | UInt16 | Сколько дней прошло от первого визита клиента до этой сессии. 0 = первый визит |
| `days_since_prev_visit` | UInt16 | Сколько дней прошло с предыдущей сессии этого клиента. **0 имеет два значения:** первый визит (нет предыдущего) ИЛИ возврат в тот же день. Для анализа интервалов возврата всегда фильтровать `WHERE visit_number > 1` |
| `goals_in_visit` | Array(UInt32) | Массив ID целей, сработавших в этой сессии. Может быть пустым. Для проверки конкретной цели: `has(goals_in_visit, GOAL_ID)` |
| `has_lead` | UInt8 | 1 = в этой сессии сработала цель 314553735 «Все лиды» |
| `has_call` | UInt8 | 1 = в этой сессии был звонок (goals 63191746, 201619840, 201619843, 201619846) |
| `has_form` | UInt8 | 1 = в этой сессии отправлена форма (goals 313904512, 314247991, 314247265) |
| `has_crm_created` | UInt8 | 1 = в этой сессии создан CRM-заказ (goal 332069613) |
| `has_crm_paid` | UInt8 | 1 = в этой сессии оплачена сделка (goal 332069614) |
| `is_converting_visit` | UInt8 | **1 = это первая сессия клиента, в которой произошёл лид** (goal 314553735). У клиента максимум одна такая строка. Используется для last-touch атрибуции по конвертирующей сессии. Отличие от `has_lead`: `has_lead = 1` может быть у нескольких сессий клиента (повторные обращения), `is_converting_visit = 1` — только у первой |

### Важные особенности

- **`days_since_prev_visit = 0`** при `visit_number = 1` — технический артефакт (нет предыдущего визита). Всегда добавлять `WHERE visit_number > 1` при анализе интервалов возврата.
- **`has_lead` vs `is_converting_visit`:** `has_lead` отражает факт лида в конкретной сессии (может быть у нескольких сессий), `is_converting_visit` помечает только первую конверсионную сессию клиента.
- **`goals_in_visit`** содержит все сработавшие цели включая поведенческие (просмотр ЖК и т.д.). Для поиска конкретной цели: `has(goals_in_visit, 402733217)` — спам, `has(goals_in_visit, 330230879)` — добавил в избранное и т.д.
- **Связь с другими витринами:** `visit_id` позволяет JOIN с `visits_all_fields` за деталями конкретной сессии. `client_id` связывает с `dm_client_profile`.

### Типовые запросы

```sql
-- На каком визите чаще всего звонят
SELECT visit_number, count() AS calls
FROM magnetto.dm_client_journey
WHERE has_call = 1
GROUP BY visit_number ORDER BY visit_number

-- Какой канал присутствует на конвертирующем визите
SELECT utm_source, traffic_source, count() AS conversions
FROM magnetto.dm_client_journey
WHERE is_converting_visit = 1
GROUP BY utm_source, traffic_source ORDER BY conversions DESC

-- Средний интервал возврата по каналам
SELECT first_utm_source, round(avg(j.days_since_prev_visit), 1) AS avg_return_days
FROM magnetto.dm_client_journey j
JOIN magnetto.dm_client_profile p ON j.client_id = p.client_id
WHERE j.visit_number > 1
GROUP BY first_utm_source ORDER BY avg_return_days
```
